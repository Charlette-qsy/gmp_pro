## Section IV GMP FPGA Verilog Standard

### 信号的命名规则

+ 由语法带来的限制

Verilog中信号量的命名仅能由字母、数字、_组成，语法中额外允许使用$作为仿真系统任务的首字符，但不应在库代码中使用。

命名的第一个字符必须为字母或_。

模块中的命名应当唯一。

+ 特殊要求

对于以下指出的信号为了使用方便，应当遵照下面的规则执行：、

| 特殊符号 | 说明             |
| -------- | ---------------- |
| clk      | 时钟输入         |
| rstn     | 复位，低电平有效 |



+ 前缀、后缀规则

为了方便区分信号的用途，从而需要规定信号的前缀和后缀。

> 前缀

| 前缀            | 说明                     |
| --------------- | ------------------------ |
| `clk_`          | 时钟信号                 |
| `rst_`或`rstn_` | 复位信号                 |
| `en_`           | 使能信号                 |
| `cs_`           | 片选信号                 |
| `param_`        | 可以修改的寄存器变量前缀 |

> 信号后缀

| 后缀 | 说明       |
| ---- | ---------- |
| `_r` | 寄存器     |
| `n`  | 低电平有效 |


+ 常数常量命名规则

应当全部使用**大写字母**，不同单词之间使用`_`作为分割。

+ 信号的基本命名规则

对于常用的单词组或者专业术语在下面表格中给出，对于其他情况应当使用全拼，各个单词之间使用`_`作为分割。

| 简写     | 全拼                                 | 说明             |
| -------- | ------------------------------------ | ---------------- |
| `cordic` | Coordinate rotation digital computer | 坐标旋转数字算法 |
| `rd`     |                                      |                  |
| `wr`     |                                      |                  |

+ 注释规则

每一个独立的功能模块（module, always, task, function）应当有简答的注释说明。

每一个输入输出引脚的目标和作用应当进行描述。

+ 复位逻辑规则

FPGA的复位逻辑应当是在接收到异步复位信号之后除了极个别的特殊情况，其他的寄存器应当处于置零状态。






## Section V GMP Simulink Library Standard

### Mask Editor Standard

+ Edit Box item

每一个编辑框需要带一个文本框说明参数的含义。文本框位于编辑框的上面。文本框的文字内容末尾没有标点，需要包含参数的物理意义和单位；编辑框的提示符内容需要写为`参数名称:`。
